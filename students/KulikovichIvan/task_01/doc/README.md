# Лабораторная работа 01. Контейнеризация и Docker

## Описание проекта

Это простое веб-приложение на Python Flask с эндпоинтами (/health) и поддержкой graceful shutdown. Проект упакован в Docker-контейнер с использованием многоуровневой сборки для удобного и эффективного развертывания на различных устройствах. Приложение имеет одну основную зависимость - клиент Redis для работы с хранилищем данных.

## Структура проекта

- `app/` - Исходный код сервера
- `Dockerfile` - Конфигурация для сборки образа приложения
- `docker-compose.yml` - Конфигурация для запуска стека сервисов
- `requirements.txt`- Файл для установки зависимостей
- `main.py`- Основной код
- `doc/` - Документация
- `README.md` - Документация проекта

## Технологии

- Docker & Docker Compose
- Redis для хранения состояния
- Веб-сервер с поддержкой graceful shutdown
- Health checks для мониторинга состояния

## Запуск проекта

### Сборка и запуск через Docker Compose

docker-compose up --build
Приложение будет доступно по адресу: http://localhost:8073

## Ручная сборка и запуск

docker build -t web-app .
docker run -p 8073:8073 web-app

## Функциональность

Счетчик посещений, сохраняемый в Redis

Graceful shutdown при получении SIGTERM

Health checks для мониторинга состояния

## Сохранение данных при перезапуске контейнеров

docker inspect web-app:stu-220015-v11 | grep -A 10 "Labels"

## Проверка graceful shutdown

Запустите приложение и выполните:
docker-compose down
В логах должно появиться сообщение о начале graceful shutdown.

## Проверка health check

docker ps
В статусе контейнера должно отображаться Up (healthy).
