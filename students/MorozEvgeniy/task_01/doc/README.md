## Инструкция по сборке и запуску

1.  **Склонируйте репозиторий:**
    ```bash
    git clone <URL_вашего_репозитория>
    cd <папка_репозитория>
    ```

2.  **Соберите образы и запустите контейнеры:**
    Выполните команду в корневой папке проекта (где находится `docker-compose.yml`):
    ```bash
    docker compose up --build
    ```
    Ключ `--build` принудительно пересобирает образ, если в `Dockerfile` или исходном коде были изменения. При последующих запусках его можно опустить.

3.  **Проверка работы сервиса:**
    * Откройте в браузере или выполните в терминале `curl http://localhost:8063`. Вы должны увидеть приветственное сообщение и счетчик посещений.
    * Проверьте эндпоинт для healthcheck: `curl http://localhost:8063/ping`. В ответ должно прийти `OK`.
    * Посмотрите логи запущенных контейнеров:
        ```bash
        docker compose logs -f app
        ```
        В логах `app` вы увидите сообщение о старте сервера и ваши метаданные.

4.  **Проверка Graceful Shutdown (корректного завершения):**
    Чтобы остановить сервисы, нажмите `Ctrl+C` в том же терминале, где вы запускали `docker compose up`, или выполните в другом терминале команду:
    ```bash
    docker compose down
    ```
    В логах сервиса `app` вы увидите сообщения о получении сигнала и успешной остановке сервера. Это и есть демонстрация graceful shutdown.
